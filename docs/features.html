The Science of Context.

ConteXia is a high-latency, multi-modal context engine disguised as a nature-themed sidepanel. Here is a technical deep-dive into its core systems.

Vision Mapping & Spatial Reasoning

Unlike standard text scrapers, ConteXia uses dynamic viewport capture to ingest "Visual Context". This allows it to understand layouts, diagrams, and non-selectable text within the user's active screen.

How it works:

  • Dynamic Viewport Ingestion Captures the active tab's layout using chrome.tabs.captureVisibleTab when 3D vision is triggered.
  • Spatial Metadata Ingests DOM coordinate data for hover triggers, allowing the AI to know exactly what the user is looking at.
  • OCR & PDF Depth Bypasses traditional PDF security by reading the visual rendered layer, ensuring 100% data access.
  • Real-time capture
  • DOM Spatial Awareness
  • PDF Layer Bypass

Tree Log: Reactive Physics Interface

The Tree Log UI is a exercise in boutique frontend engineering, combining 9-slice tiling with reactive flexbox logic to create a container that feels physical and weighted.

How it works:

  • 9-Slice Bark Texture Uses a custom-engineered border-image slice to scale the bark texture without pixel stretching, regardless of sidebar width.
  • Vertical Elasticity The input box uses Svelte reactive bindings to auto-grow up to a 4-line threshold before enabling a custom-styled scrollbar-wood.
  • Micro-interaction Engine Icons and tools shift dynamically to the corners of the "carved plank" to maximize typing real-estate.
  • Elastic Input Height
  • 9-Slice Non-distort Scaling
  • Wood-textured Scrolling

Presence: Ethereal Audio Dialogue

Presence mode transforms the assistant into a persistent, hands-free presence. It leverages a high-speed STT/TTS pipeline for ultra-low latency human-grade conversation.

How it works:

  • Gooey Blob Physics Uses SVG filters and GSAP to animate "liquid" blobs that respond to real-time AudioContext frequency data.
  • Interruptible Pipeline Implements a "barge-in" mechanism where incoming mic activity immediately halts current TTS playback to permit natural turn-taking.
  • Latency Triage Distributes workloads between Groq (STT/LLM) and Cartesia (TTS) to maintain a response latency sub-500ms.
  • Sub-500ms Audio Latency
  • Interruptible TTS Playback
  • Frequency-Reactive Physics

Boutique Annotation Ink

The Annotation Overlay is a specialized canvas layer that injects directly into the host page, allowing for persistent, non-destructive thought mapping.

How it works:

  • Shadow DOM Isolation Injects the canvas into a Shadow Root to prevent host page CSS from leaking into the drawing engine.
  • Ink Physics Uses quadratic curve interpolation to smooth out rapid mouse movements into elegant "stationery" strokes.
  • State Persistence Annotations are serialized and hashed to the site URL, ensuring they remain visible when you revisit a page (via chrome.storage).
  • Quadratic Stroke Smoothing
  • Shadow DOM Protection
  • URL-Persistent State

Dwell Intelligence (Proactive Assistance)

ConteXia implements a silent "Reading Sentinel" that monitors the balance between content density and user engagement to offer help exactly when needed.

How it works:

  • Focus Tracking Monitors scroll and mouseover dwell-time on specific paragraphs.
  • Density Thresholding Triggers a "Smart Hint" chime when a high-density text section results in a dwell-time outlier (>2x average).
  • Invisible Ingestion Prefetches context in small chunks to ensure immediate summarization if the user eventually selects the text.
  • Dwell-time Anomaly Detection
  • Proactive Audio Chimes
  • Context Prefetching

Production-Grade Safety & Reliability

ConteXia is built for reliability. It includes robust back-end safeguards to prevent abuse and ensure high availability in production-like environments.

How it works:

  • Token-Aware Rate Limiting Implements a persistent rate limiter in chrome.storage.local to enforce RPM quotas (10 RPM for LLMs, 20 RPM for TTS).
  • Fallback Triage Automatically switches between Gemini and Groq depending on quota availability and response health.
  • Secure Key Management All API keys are encrypted at rest and never shared across instances.
  • Persistent Rate Limiting
  • Multi-Model Fallbacks
  • End-to-End Asset Cleanup
